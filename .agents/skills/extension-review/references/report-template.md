## Markdown Report Template

````markdown
# Chrome Web Store Review Report

**Extension**: {name}
**Version**: {version}
**Scanned**: {date}
**Verdict**: {LIKELY APPROVED | NEEDS FIXES | WILL BE REJECTED}

## Summary

| Severity | Count |
|----------|-------|
| CRITICAL | {n}   |
| HIGH     | {n}   |
| MEDIUM   | {n}   |
| PASS     | {n}   |

---

## Issues Found

### {Issue Name}

| Field | Detail |
|-------|--------|
| **Severity** | CRITICAL / HIGH / MEDIUM |
| **Code** | {Chrome violation code, e.g. Blue Argon} |
| **Root Cause** | {Plain English explanation of what triggers this} |
| **File(s)** | `{path/to/file.ts}:{line}` |

**What's wrong:**
{1-2 sentence explanation of the problem}

**How to fix:**
1. {Step 1}
2. {Step 2}
3. {Step 3}

---

{Repeat for each issue}

## Passed Checks

- [x] No remote code execution
- [x] No code obfuscation
- [x] Permissions match usage
- [x] Privacy policy present
{... list all passing checks}

## Next Steps

1. Fix all CRITICAL issues first
2. Address HIGH issues
3. Review MEDIUM issues
4. Re-run this scan to verify fixes
5. Submit to Chrome Web Store
````

## Verdict Logic

- **WILL BE REJECTED**: Any CRITICAL issue found
- **NEEDS FIXES**: 1+ HIGH issues, no CRITICAL
- **LIKELY APPROVED**: Only MEDIUM or no issues

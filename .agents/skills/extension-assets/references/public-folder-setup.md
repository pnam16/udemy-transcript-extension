# Public Folder Setup for Chrome Extensions

## Directory Structure

```
public/
├── icons/
│   ├── icon16.png        # Extension icon 16x16
│   ├── icon32.png        # Extension icon 32x32
│   ├── icon48.png        # Extension icon 48x48
│   ├── icon128.png       # Extension icon 128x128 (also CWS store icon)
│   ├── action16.png      # Toolbar action icon 16x16
│   ├── action24.png      # Toolbar action icon 24x24
│   └── action32.png      # Toolbar action icon 32x32
├── store/
│   ├── screenshot1.png   # CWS screenshot (1280x800)
│   ├── screenshot2.png
│   ├── promo-small.png   # Small promo tile (440x280)
│   └── promo-large.png   # Large promo tile (920x680) — optional
├── _locales/             # i18n strings (optional)
│   ├── en/
│   │   └── messages.json
│   └── es/
│       └── messages.json
└── manifest.json         # Only if not generated by build tool
```

Note: `store/` assets are NOT included in the extension package — upload directly to CWS dashboard.

## Required Files

```bash
# Create directory structure
mkdir -p public/icons public/store public/_locales/en

# Placeholder for empty dirs (git tracking)
touch public/_locales/en/.gitkeep
```

## manifest.json Icon Paths

```json
{
  "manifest_version": 3,
  "icons": {
    "16": "icons/icon16.png",
    "32": "icons/icon32.png",
    "48": "icons/icon48.png",
    "128": "icons/icon128.png"
  },
  "action": {
    "default_icon": {
      "16": "icons/action16.png",
      "24": "icons/action24.png",
      "32": "icons/action32.png"
    },
    "default_popup": "popup.html"
  }
}
```

Paths are relative to extension root (where manifest.json lives), not project root.

## Web Accessible Resources

Declare any public assets accessed by content scripts:

```json
{
  "web_accessible_resources": [{
    "resources": ["icons/*.png", "assets/*.css"],
    "matches": ["<all_urls>"]
  }]
}
```

## i18n messages.json Template

```json
{
  "appName": {
    "message": "My Extension",
    "description": "Extension name shown in Chrome"
  },
  "appDescription": {
    "message": "What my extension does",
    "description": "Extension description"
  }
}
```

Reference in manifest: `"name": "__MSG_appName__"`

## Asset Optimization

```bash
# PNG compression (install: brew install pngquant)
pngquant --quality=85-95 --force --output public/icons/icon128.png public/icons/icon128.png

# Batch compress all icons
pngquant --quality=85-95 --ext .png --force public/icons/*.png

# Verify file sizes
ls -lh public/icons/
```

## Build Step Integration

### Vite/webpack — copy public/ to dist/

Vite copies `public/` automatically. For webpack:

```js
// webpack.config.js
const CopyPlugin = require('copy-webpack-plugin')
module.exports = {
  plugins: [
    new CopyPlugin({ patterns: [{ from: 'public', to: '.' }] })
  ]
}
```

### Verify build output

```bash
# After build, confirm icons copied
ls dist/icons/
# Should show: icon16.png icon32.png icon48.png icon128.png action*.png
```

## Validation Script

```bash
#!/bin/bash
# validate-assets.sh — run before publishing
ERRORS=0
for size in 16 32 48 128; do
  FILE="public/icons/icon${size}.png"
  if [ ! -f "$FILE" ]; then
    echo "MISSING: $FILE"; ERRORS=$((ERRORS+1))
  else
    DIM=$(identify -format "%wx%h" "$FILE")
    [ "$DIM" = "${size}x${size}" ] && echo "OK: $FILE" || echo "WRONG SIZE: $FILE ($DIM)"; ERRORS=$((ERRORS+1))
  fi
done
[ $ERRORS -eq 0 ] && echo "All icons valid" || echo "$ERRORS error(s) found"
```
